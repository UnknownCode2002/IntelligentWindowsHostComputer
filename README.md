# 智能窗户2.0

本项目为 Windows 平台的桌面应用程序（基于 WinForms），用于控制与管理智能窗户设备。该仓库包含 UI 界面、设备控制逻辑以及相关资源与第三方依赖包。

## 主要功能
- 窗户开合控制与状态显示
- 设备添加/移除与配置管理
- 本地界面控制与日志记录

## 技术栈
- 开发平台：.NET Framework / Windows Forms
- 语言：C#
- 依赖管理：NuGet（本仓库包含 `packages/` 目录的离线包）

## 系统要求
- 操作系统：Windows 7 及以上(建议Windows11,其余版本UI显示可能存在兼容问题)
- 运行时：.NET Framework 4.6.1 或更高（建议使用 Visual Studio 自带的目标框架）

## 快速开始（在开发环境中）
1. 使用 Visual Studio 打开解决方案：`智能窗户2.0.sln`。
2. 还原 NuGet 包（如果使用在线源，则选择还原；仓库内已有 `packages/` 目录可作为离线依赖）。
3. 选择启动项目 `智能窗户2.0`，设置为 Debug 或 Release 配置。
4. 运行（F5）即可启动应用程序。

## 项目结构（简要）
- `智能窗户2.0.sln`：解决方案文件
- `智能窗户2.0/`：主项目文件夹
  - `mainForm.cs`：主窗体代码
  - `device.cs`：设备逻辑相关代码
  - 其他窗体与控件代码（如 `control.cs`, `roundButton.cs` 等）
- `packages/`：NuGet 离线包（第三方库）
- `bin/`, `obj/`：编译输出与中间文件

## 构建与调试注意事项
- 如果遇到缺少引用或目标框架不匹配，请在项目属性中检查目标框架并安装相应的 .NET Framework。
- 若使用不同版本的 Visual Studio，建议先在本机还原 NuGet 包并确保所有第三方库能成功加载。

## 代码修改建议
- 修改 UI：在对应的 `.Designer.cs` 文件与窗体代码中进行，保持资源文件（`.resx`）同步修改。
- 修改设备逻辑：主要在 `device.cs` 中维护设备状态与控制命令。
